# v0.5 Authentication

## Goal

Move GlucoPlate from browser-local usage toward real user accounts and cloud-backed personalization.

## Status

In progress

## User outcomes

- Users can sign in with Google, Apple, email, or guest mode.
- Saved recipes follow the user across devices.
- Notification tokens are tied to authenticated users.
- Profiles can support family or household preferences.
- Enterprise users are enrolled by access code and appear in the administrator directory.

## Implementation notes

Firebase Authentication is the identity provider. Recipe data remains under `app_data/enterprises/{enterprise_id}/users/{firebase_uid}`, while administrator membership data is maintained under `enterprise_directory`. The Firebase session endpoint performs an idempotent directory upsert so existing authenticated users are backfilled on their next session.

## Release notes draft

GlucoPlate supports Firebase sessions with Google and email/password sign-in, backend ID-token verification, cloud-backed user recipe data, enterprise access-code enrollment, and synchronized administrator directory membership. Apple sign-in, household profiles, notification ownership, and final end-to-end coverage remain before the milestone is complete.
